# Sunrise Tech Check 0.1.8

## What's new

- **Traceroute removed.** Forward and reverse traceroute didn't prove useful in practice — the data was noisy past the first few hops and rarely helped diagnose the network issues the tool is actually meant to surface. Removing it shortens the run and shrinks the surface of what the tool does on your machine.

## Internal

- Removed `internal/icmptrace` package, `cmd/techcheck/internal/nettest/traceroute.go`, the beacon's `/preflight/traceroute-back` handler, and the `Traceroute` field on each `POPResult` in the report schema.
- Reports uploaded by 0.1.8 no longer include the `traceroute` key on per-POP results. The ingest service treats it as optional.
- `wails.json` `productVersion` bumped from 0.1.6 to 0.1.8 — it was out of sync after 0.1.7.
- Beacon Ansible role: removed `AmbientCapabilities=CAP_NET_RAW` / `CapabilityBoundingSet=CAP_NET_RAW` from the systemd unit (they existed only to support raw-socket ICMP for icmptrace), and removed the `net.ipv4.ping_group_range` sysctl tweak. After redeploy, `sudo rm /etc/sysctl.d/99-beacon-icmp.conf` on each beacon host to drop the stale config file (the sysctl reverts to default on next reboot).
