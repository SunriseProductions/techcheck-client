# Sunrise Tech Check 0.1.9

## What's new

- **Rotated upload credentials.** The bearer token used by Tech Check to upload reports to the ingest service has been rotated. 0.1.9 binaries carry the new token. After all clients have updated, the previous token is revoked server-side.

## Internal

- The upload token, ingest URL, IT contact email, and fallback POP list are no longer embedded in source as JSON. They are injected at build time via Go `-ldflags`, sourced from `private/secrets.env` (gitignored) in the Makefile.
- `cmd/techcheck/internal/config/defaults/defaults.json` now holds only non-secret tunables (timeouts, retries, UDP magic).
- `cmd/techcheck/internal/config/defaults/pops.json` deleted; fallback POP list comes from `defaults.FallbackPOPsJSON` (ldflags string of a JSON array, parsed at startup).
- New `cmd/techcheck/internal/config/defaults` Go subpackage holds the ldflags-overridable variables.
- `config.Load()` applies precedence: embedded tunables → ldflags overrides → sidecar JSON.
- Public builds (no `private/secrets.env`) compile and run, but uploads fail without auth. The tool still produces and saves a local report — useful for contributors building from source.
