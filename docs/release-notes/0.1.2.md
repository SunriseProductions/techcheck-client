## What's new in 0.1.2

### Faster per-POP tests

Forward and reverse traceroute now run concurrently for each POP instead of one after the other. Saves roughly 30–50 seconds per POP on networks where the trace budget used to be fully spent.

### Honest throughput errors

A failed download test used to surface a bare `context deadline exceeded` line. It now tells you how much was actually transferred and at what effective rate, for example:

> `download stalled at 3.2 Mbps after 15s (received 6.0 MB of 10 MB): context deadline exceeded`

This makes it obvious when a venue's link is genuinely too slow for the test versus when something else intervened (VPN, middlebox, proxy).

### Graceful handling when a POP is busy

If several machines happen to run Tech Check against the same POP at the same time, the POP now refuses the third simultaneous bandwidth test with a polite "retry shortly" response. The desktop app honours that and retries transparently. Crucially, the retry wait is **not** counted toward the reported Mbps — the number you see reflects only the successful transfer, so you don't get a false "slow network" reading because another laptop was testing at the same instant.

### Fixes

- Minor stability improvements around malformed server responses.
