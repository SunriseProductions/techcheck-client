# Sunrise Tech Check 0.1.4

## What's new

- **Traceroute now works on Windows in environments where `tracert.exe` was failing** (notably some EC2 / restricted-egress setups) — we no longer shell out to the system `tracert` / `traceroute` binaries.
- **No more cmd window flashes during the run on Windows.** Replaced shell-out to `tracert.exe` with a Go-native ICMP traceroute that uses `IcmpSendEcho2` from `iphlpapi.dll` (the same Windows kernel helper `tracert.exe` itself uses internally), and suppressed the console window on the remaining PowerShell calls used for AV-product and firewall-status detection.
- **Same Go-native traceroute on macOS and Linux** — uses an unprivileged DGRAM ICMP socket. No more shelling out to the system `traceroute` binary.

## Internal

- New `internal/icmptrace` package, build-tag-split between `IcmpSendEcho2` (Windows) and `golang.org/x/net/icmp` (Linux/macOS).
- Beacon `/preflight/traceroute-back` now uses the same package — consistent traceroute behaviour client-side and server-side.
