# Sunrise Tech Check 0.1.10

## What's new

- **Signed and notarised by Apple.** macOS no longer shows the "unidentified developer" warning when you first open Tech Check. The binary is signed under Sunrise's Developer ID and notarised by Apple, so Gatekeeper just lets it run.
- **Minimum macOS version: 10.15 Catalina.** Notarisation requires the hardened runtime, which needs macOS 10.15 or later. If you're on anything older (very unlikely on a Sunrise machine), the auto-updater won't offer you 0.1.10 — stay on 0.1.9.

## Internal

- Hardened runtime enabled at build time (`codesign --options runtime`) with an `entitlements.plist` declaring `com.apple.security.network.client` — the only entitlement the app actually needs.
- Bundle identifier changed from the Wails default `com.wails.SunriseTechCheck` to `productions.sunrise.sunrise-techcheck`. Wails' default would not have notarised cleanly because it doesn't match the signing team's domain conventions.
- `LSMinimumSystemVersion` raised from 10.13.0 to 10.15.0 in `cmd/techcheck/build/darwin/Info.plist`.
- New Makefile targets in the public client repo: `sign-mac-*`, `verify-sign-mac`, `notarize-mac-*`, `release-desktop`. `dist-mac-*` now depends on `sign-mac-*`, so even `make build-desktop` produces signed (ad-hoc or real) artifacts.
- New `APPLE_SIGN_IDENTITY`, `APPLE_ID`, `APPLE_TEAM_ID`, `APPLE_APP_PASSWORD` environment variables, sourced from `private/secrets.env` and passed through from the integration repo's Makefile.
- Public CI fixes: e2e test no longer hits the WaitDelay flake (mocks pre-built in `TestMain` and launched directly so `cmd.Process.Kill()` actually kills them rather than orphaning them under a `go run` wrapper); workflow now runs `wails build` from `cmd/techcheck/` where `wails.json` lives; Linux CI builds with `-tags webkit2_41` to match ubuntu-24.04's GTK packages.
